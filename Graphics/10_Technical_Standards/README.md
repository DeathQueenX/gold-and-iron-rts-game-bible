# 10 — Technical Standards

## Scop

Stabilirea regulilor tehnice comune pentru ca asset-urile să poată fi create, revizuite și importate în Unity fără inconsistențe.

## Standarde de definit

- unitatea de măsură și scara proiectului;
- orientarea axelor în Blender și Unity;
- originea și pivotul obiectelor;
- convenții de denumire;
- structură de directoare;
- formate sursă și formate de export;
- reguli FBX;
- rezoluții de texturi;
- PBR și canale de materiale;
- atlasuri și trim sheets;
- buget de poligoane;
- LOD-uri;
- rigging și animații;
- coliziuni;
- setări de import Unity;
- prefabs și variante;
- versionare și reguli de aprobare.

## Convenție inițială de naming

Format recomandat:

`Categorie_Factiune_Nume_Varianta_Tip`

Exemple:

- `UNT_Ferrum_Swordsman_A_Model`
- `BLD_Aurelia_Barracks_A_Model`
- `VFX_Thalassia_TidalStrike_A_Prefab`
- `UI_Global_HealthBar_A_Sprite`

Convenția finală va fi sincronizată cu structura proiectului Unity.
